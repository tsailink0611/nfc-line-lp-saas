# NFC x LINE x LP 営業支援SaaS MVP — プロジェクト概要書

## 1. システム概要

NFCタグをタッチすると営業担当者の専用ランディングページ（LP）が表示され、
会社支給LINEへの友達追加を誘導する営業支援システム。

**フロー**: NFCタッチ → 担当者LP表示 → LINE友達追加 → 商談へ

### 本番URL
| ページ | URL |
|--------|-----|
| 担当者LP（例） | `${NEXT_PUBLIC_SITE_URL}/staff/{slug}` |
| NFCリダイレクト（例） | `${NEXT_PUBLIC_SITE_URL}/n/{token}` |
| 管理画面 | `${NEXT_PUBLIC_SITE_URL}/admin/login` |

## 3. 管理画面の機能一覧

### 3-1. ダッシュボード
- 担当者数、公開中数、店舗数、キャンペーン数の統計カード
- 最近の更新一覧

### 3-2. 担当者管理（CRUD）
- **登録項目**: 姓名、英語名、表示名、スラッグ（URL用）、店舗、役職、キャリア年数、プロフィール文、得意分野、メイン画像、サブ画像、LINE追加URL、会社支給携帯番号、YouTube URL、表示順、公開状態
- **バッジ管理**: 担当者ごとにスキルバッジ（例: 新築マンション、土地活用）を追加・削除
- **ギャラリー管理**: 画像を複数アップロード可能
- **公開/非公開の切替**: トグルで即時切替

### 3-3. 店舗管理（CRUD）
- 店舗名、店舗コード、住所、郵便番号、電話番号、営業時間、定休日、Google Map埋め込みURL

### 3-4. キャンペーン管理（CRUD）
- タイトル、概要、画像、リンクURL、適用店舗、開始日・終了日、公開状態
- 自動で「開催中」「受付中」バッジ表示

### 3-5. NFC管理
- NFCトークンの発行（ランダム文字列を自動生成）
- 担当者との紐付け
- リダイレクト先パスの設定
- 有効/無効の切替、削除

### 3-6. 設定
- **会社情報**: 会社名（日英）、ロゴ、メインカラー、サブカラー、会社説明
- **LP設定**: ヒーローキャッチコピー、サブキャッチ、CTA文言、フッター文言、テーマ選択

## 4. NFCカードの運用フロー

```
1. 管理画面 > NFC管理 > トークン発行
2. リダイレクト先を担当者のLP（/staff/{slug}）に設定
3. NFCカードに書き込むURL:
   ${NEXT_PUBLIC_SITE_URL}/n/{token}
4. 見込み客がNFCタッチ → 自動的に担当者LP へリダイレクト
5. LP上の「LINEで気軽に相談する」ボタン → LINE友達追加
```

## 5. セキュリティ評価

### 現状の対策（実装済み）

| 対策 | 状態 | 詳細 |
|------|------|------|
| 認証 | 済 | Supabase Auth（email/password）、JWT Cookie ベース |
| 認証ガード | 済 | Next.js Middleware で /admin/* を保護（未認証→ログインリダイレクト） |
| RLS（行レベルセキュリティ） | 済 | 全9テーブルにRLS有効。company_id ベースでデータ分離 |
| 管理者権限チェック | 済 | `is_admin_of(company_id)` ヘルパー関数で一元管理 |
| 入力バリデーション | 済 | Zod で全フォームをサーバーサイドバリデーション |
| 画像アップロード制限 | 済 | MIME チェック + 5MB 上限 |
| HTTPS | 済 | Cloudflare Workers は全てHTTPS |
| XSS対策 | 済 | React のデフォルトエスケープ + Server Components |

### 要改善項目（優先度順）

| # | 項目 | リスク | 対応方法 |
|---|------|--------|----------|
| 1 | **CSRF対策** | 中 | Server Actions はデフォルトでPOSTだが、明示的なCSRFトークン未実装。Next.js 15のServer Actionsは内部的にCSRF保護があるため実質的には低リスク |
| 2 | **レート制限** | 中 | ログイン試行の回数制限なし。Cloudflare WAFまたはSupabase側のrate limitで対応可能 |
| 3 | **パスワードポリシー** | 低 | 現状は任意の文字列。Supabase Auth設定で最低文字数を設定推奨 |
| 4 | **監査ログ** | 低 | 誰がいつ何を変更したかのログなし。将来的にaudit_logsテーブル追加で対応 |
| 5 | **環境変数のハードコード** | なし | 全てenv経由。問題なし |

### セキュリティ総合評価
**MVP段階としては十分**。RLSによるマルチテナント分離とSupabase Authの組み合わせは堅牢。
本番運用開始前にレート制限の追加を推奨。

## 6. 1人10社運用モデル（月額15万円/社）

### 6-1. ビジネスモデル

```
月売上: 15万円 x 10社 = 150万円
原価:
  - Supabase Pro: $25/月（約4,000円）— 全社共有
  - Cloudflare Workers: 無料枠で十分（10万リクエスト/日）
  - NFCカード: 1枚300-500円（初期のみ）
  - 自分の工数: 月40-60時間
粗利: 約145万円
```

### 6-2. 現状のアーキテクチャでの課題

現在は **1社 = 1Supabaseプロジェクト + 1 Workers** の構成。
10社運用するには以下の選択肢がある:

| 方式 | メリット | デメリット |
|------|---------|-----------|
| **A: 1社1デプロイ（現状）** | データ完全分離、障害影響なし | 10個のプロジェクト管理が煩雑 |
| **B: マルチテナント化** | 1つの管理画面で全社管理 | 開発工数が必要（2-3日） |

**推奨: B（マルチテナント化）**
- 既にcompany_idが全テーブルにある
- RLSもcompany_id ベースで実装済み
- 必要な追加開発:
  1. 「スーパー管理者」ロール追加（全社横断で操作可能）
  2. 管理画面に「会社切替」UI追加
  3. 会社ごとのサブドメインまたはパス分岐

### 6-3. 1人で10社回す運用フロー

#### 月次サイクル（1社あたり約4-6時間/月）

```
【初期導入（1回のみ）: 約8時間/社】
1. ヒアリング（2h）: 会社情報、カラー、営業担当者リスト、LINE URL
2. セットアップ（3h）: 管理画面で会社・店舗・担当者を登録、LP調整
3. NFCカード手配・設定（1h）: NFCにURL書込み、動作確認
4. 導入説明会（2h）: 営業担当者向けにNFCの使い方をレクチャー

【月次運用: 約2-4時間/社】
Week 1: 管理画面で変更依頼対応（担当者追加/退職対応、キャンペーン更新）
Week 2-3: 放置（自動運用）
Week 4: 月次ミーティング（2h）— 実績報告 + 次月キャンペーン計画

【随時対応: 月1-2時間/社】
- チャット（LINE or Chatwork）での質問対応
- 担当者の写真差替え等の軽微な変更
```

#### 10社の時間配分

| 作業 | 時間/社/月 | 10社合計 |
|------|----------|---------|
| 月次ミーティング | 2h | 20h |
| 変更対応 | 1.5h | 15h |
| 質問対応 | 0.5h | 5h |
| **合計** | **4h** | **40h** |

**1日2時間で10社運用可能。**

### 6-4. 効率化のために追加すべき機能

| # | 機能 | 効果 | 開発工数 |
|---|------|------|---------|
| 1 | **スーパー管理者ダッシュボード** | 全社の状況を1画面で把握 | 1-2日 |
| 2 | **テンプレート機能** | 新規会社の初期設定を数クリックで完了 | 1日 |
| 3 | **一括CSV取込** | 営業担当者30名を一括登録 | 0.5日 |
| 4 | **アクセス解析** | NFCタップ数、LP閲覧数、LINEクリック数を可視化 | 2-3日 |
| 5 | **月次レポート自動生成** | PDF or メールで自動送信 | 2日 |
| 6 | **Webhook通知** | NFCタップ時にSlack/LINE通知 | 0.5日 |

## 7. 現状の課題と改善点

### 7-1. UX改善

| # | 課題 | 詳細 | 優先度 |
|---|------|------|--------|
| 1 | LP プレビュー機能なし | 保存後に別タブで確認する必要がある | 高 |
| 2 | 画像リサイズ未対応 | アップロード時に自動リサイズされない（大きい画像は表示が遅い） | 中 |
| 3 | ドラッグ&ドロップ並び替え | 担当者・バッジの表示順をD&Dで変更できない | 中 |
| 4 | 一括操作 | 複数担当者の公開/非公開を一括変更できない | 低 |
| 5 | モバイル管理画面 | 管理画面のモバイル対応はあるが、操作性は改善余地あり | 低 |

### 7-2. 機能拡張ロードマップ

```
Phase 1（現在）: MVP — 1社運用可能
Phase 2: マルチテナント化 + アクセス解析
Phase 3: 月次レポート自動生成 + テンプレート
Phase 4: LINE Messaging API連携（実際の友達追加トラッキング）
Phase 5: 営業担当者向け簡易ダッシュボード（自分のLP成果を確認）
```
